## Snowflake MCP Server

Required for both Cursor (Developers) and Claude (Business Users).

### For Developers (Cursor)
1.  Open Cursor Settings > **Features** > **MCP**.
2.  Add a new MCP server:
    *   **Type**: `stdio` (or Docker, depending on deployment)
    *   **Command**: Command to run the Snowflake MCP connector.
    *   **Environment Variables**:
        *   `SNOWFLAKE_ACCOUNT`
        *   `SNOWFLAKE_USER`
        *   `SNOWFLAKE_PASSWORD` (or authenticator configuration)
        *   `SNOWFLAKE_ROLE`
        *   `SNOWFLAKE_WAREHOUSE`

### For Business Users (Claude.ai)
1.  Access the **Integrations** or **MCP** menu in Claude.
2.  Connect to the corporate Snowflake instance.
3.  Authenticate using your SSO credentials.

## Prime (Custom Context)

**Prime** is our custom MCP server that indexes dbt manifest files.

*   **Location**: Hosted internally.
*   **Connection**: Developers connect via Cursor MCP settings pointing to the Prime endpoint.
*   **Usage**: Automatically provides context on `dbt` models, tests, and lineage to the AI model.

## Lightdash

*   Access via SSO at the company portal.
*   No local configuration required for consumers.
